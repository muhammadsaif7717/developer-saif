## 2. 🎨 Design System

### Theme Philosophy
**Dual-mode experience designed for developer aesthetic with premium feel**

#### 🌙 Dark Mode (Primary/Default)
- **Mood:** Sophisticated, focused, cutting-edge
- **Base Colors:**
  - Background: `#000000` (pure black)
  - Card Surface: `#11141c` (dark charcoal)
  - Border: `#27273a` (subtle dividers)
  
- **Accent Palette:**
  - Primary: `#0082c4` (cyan blue) - CTAs, links, highlights
  - Secondary: `#0099e6` (lighter cyan) - hover states, accents
  - Success: `#10b981` (emerald) - positive states
  - Warning: `#f59e0b` (amber) - attention
  
- **Text Colors:**
  - Primary: `#0082c4` (cyan blue) - headings, important text
  - Secondary: `#cbd5e1` (cool gray) - body text
  - Muted: `#64748b` (slate) - captions, meta info

#### ☀️ Light Mode (Alternative)
- **Mood:** Clean, professional, approachable
- **Base Colors:**
  - Background: `#ffffff` (pure white)
  - Card Surface: `#f2f2f2` (light gray)
  - Border: `#e2e8f0` (light dividers)
  
- **Accent Palette:**
  - Primary: `#0082c4` (cyan blue) - maintains brand consistency
  - Secondary: `#006a9e` (deeper cyan) - hover states
  - Success: `#059669` (forest green)
  - Warning: `#d97706` (darker amber)
  
- **Text Colors:**
  - Primary: `#0082c4` (cyan blue) - headings, important text
  - Secondary: `#334155` (slate gray) - body
  - Muted: `#64748b` (medium slate) - captions

### Typography System
- **Font Stack:**
  - Headings: `Inter` (700, 600) - modern, geometric
  - Body: `Inter` (400, 500) - excellent readability
  - Code/Monospace: `JetBrains Mono` - for technical elements
  
- **Scale (Tailwind):**
  - Hero: `text-5xl md:text-7xl` (48px → 72px)
  - H1: `text-4xl md:text-5xl` 
  - H2: `text-3xl md:text-4xl`
  - H3: `text-2xl md:text-3xl`
  - Body: `text-base md:text-lg` (16px → 18px)
  - Small: `text-sm` (14px)

### Color Usage Guidelines
- **Primary Cyan (#0082c4):**
  - Brand logo and identity

- **Page Backgrounds:**
  - Dark mode: `#000000` (pure black) for maximum contrast
  - Light mode: `#ffffff` (pure white) for clean look

- **Card Radius:** `rounded-xl` (12px) for modern feel

### Visual Effects

## 3. 🎬 Animation System

### Animation Strategy
Animations enhance UX without overwhelming - **purposeful, smooth, performant**

### Framer Motion Implementations

#### **1. Page Transitions**
```typescript
// Fade in + slide up on route change
initial: { opacity: 0, y: 20 }
animate: { opacity: 1, y: 0 }
transition: { duration: 0.5, ease: "easeOut" }
```

#### **2. Scroll-Triggered Animations**
- **Hero Section:** Staggered fade-in for heading → subtitle → CTA
- **Services Cards:** Fade + slide up with stagger delay (0.1s each)
- **Portfolio Items:** Scale + fade on scroll intersection
- **Stats/Numbers:** Count-up animation when visible

#### **3. Hover Interactions**
- **Buttons:** Scale(1.05) + cyan glow effect
- **Project Cards:** Lift (translateY: -8px) + shadow increase + cyan border glow
- **Navigation Links:** Cyan underline slide animation
- **Social Icons:** Scale + cyan color shift

#### **4. Micro-Interactions**
- **Theme Toggle:** Smooth rotation + color morph (sun ↔ moon icon)
- **Form Inputs:** Focus scale + cyan border glow
- **Loading States:** Skeleton shimmer + pulse
- **Success/Error:** Bounce + slide notifications (sonner)

### CSS Animation Classes (tw-animate-css)
- `animate-fade-in` - Content reveals
- `animate-slide-in-bottom` - Section entries
- `animate-pulse` - Loading indicators
- `animate-bounce` - Call-to-action emphasis

### Performance Considerations
- Use `will-change` sparingly
- Leverage GPU acceleration (`transform`, `opacity`)
- Intersection Observer for scroll animations
- Disable animations on `prefers-reduced-motion`

## 5. 📱 Responsiveness & Accessibility

### Responsive Breakpoints (Tailwind)
```
sm:  640px  (mobile landscape)
md:  768px  (tablet)
lg:  1024px (desktop)
xl:  1280px (large desktop)
2xl: 1536px (wide screens)
```

### Mobile-First Strategy
- Base styles for mobile (320px+)
- Progressive enhancement for larger screens
- Touch-friendly interactive elements (min 44px tap targets)
- Optimized images with `next/image` (responsive, lazy loading)

### Accessibility Standards (WCAG 2.1 AA)

#### **Keyboard Navigation**
- All interactive elements focusable via Tab
- Visible focus indicators with cyan ring
- Skip-to-content link for screen readers
- Esc key closes modals/menus

#### **Screen Reader Support**
- Semantic HTML5 elements (`<nav>`, `<main>`, `<article>`)
- ARIA labels for icon-only buttons
- Alt text for all images
- `aria-live` regions for dynamic content (notifications)

#### **Color Contrast**
- Cyan (#0082c4) on black: 8.59:1 (AAA compliant)
- Cyan (#0082c4) on white: 4.86:1 (AA compliant)
- Card backgrounds provide sufficient contrast
- Text: Minimum 4.5:1 ratio

### Performance Optimization

#### **Core Web Vitals Targets**
- **LCP (Largest Contentful Paint):** < 2.5s
- **FID (First Input Delay):** < 100ms
- **CLS (Cumulative Layout Shift):** < 0.1

#### **Optimization Strategies**
1. **Image Optimization:**
   - WebP/AVIF formats with fallbacks
   - Responsive images via `next/image`
   - Lazy loading below the fold

2. **Code Splitting:**
   - Dynamic imports for heavy components
   - Route-based chunking (App Router automatic)
   - Tree-shaking unused code

3. **Caching Strategy:**
   - Static generation for portfolio pages
   - ISR (Incremental Static Regeneration) for dynamic content
   - CDN caching headers
